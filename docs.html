All programs must start with START (or BEGIN) and end with STOP (or END). The compiler will check your entire code for syntax errors before running it.

Comments

Comments explain code and are ignored by the compiler. Use //, #, --, or COMMENT for single lines. Use /* ... */ for multi-line blocks.

Example 1: Basic Program

START
    // This is the main program block
    PRINT "Hello, World!"
STOP

Example 2: All Comment Types

START
    // Single-line comment
    # Another single-line
    -- Yet another
    COMMENT This is also a comment
    
    /*
      This is a
      multi-line comment block.
    */
STOP

Example 3: Simple Logic

START
    // Declare and assign a variable
    INIT x = 10
    PRINT "The value of x is: ", x
STOP

Variables store data. They must be created with DECLARE or INIT before being used. Values are changed with SET, LET, or =.

Data Types

Data types are inferred automatically. The compiler supports:

  • Number: 10, 5.5
  • String: "Hello", "123"
  • Character: 'A', 'z'
  • Boolean: TRUE or FALSE
  • Null: An empty/uninitialized value.

Example 1: DECLARE (Initialize as null)

START
    DECLARE name // 'name' is now null
    PRINT "Enter name:"
    GET name
    PRINT "Hello, " + name
STOP

Example 2: INIT (Declare and Assign)

START
    INIT age = 25
    INIT message = "Hello"
    INIT isActive = TRUE
    
    // Multiple declarations on one line
    DECLARE x, y, z
    INIT a = 1, b = 2
STOP

Example 3: SET / LET (Re-Assignment)

START
    INIT score = 0
    PRINT score  // 0
    
    SET score = 10
    PRINT score  // 10
    
    LET score = score + 5
    PRINT score  // 15
    
    score = 20  // Direct assignment also works
    PRINT score  // 20
STOP
Warning: Using undeclared variables will throw an error. Always use DECLARE or INIT before use.

Output (PRINT, DISPLAY, OUTPUT)

Displays values to the terminal. All three keywords work identically. You can combine multiple items with + or commas.

Example 1: Basic Output

START
    PRINT "Hello World"
    OUTPUT 12345
    DISPLAY TRUE
STOP

Example 2: Concatenation (Joining)

START
    INIT name = "User"
    INIT age = 21
    
    // Use + to join strings
    PRINT "Hello, " + name + "!"
    
    // Use commas to print multiple items
    PRINT "You are ", age, " years old."
STOP

Example 3: Printing Calculations

START
    INIT x = 10
    INIT y = 5
    PRINT "10 + 5 = ", x + y
    PRINT "10 * 5 = ", 10 * 5
STOP

Input (INPUT, READ, GET)

Reads user input from the terminal and stores it in a declared variable. All three keywords work identically. You can optionally provide a prompt string.

Example 1: Simple Input

START
    DECLARE age
    PRINT "Enter your age:"
    INPUT age
    PRINT "You are " + age
STOP

Example 2: Input with a Prompt

START
    DECLARE name
    READ "What is your name?", name
    PRINT "Hello " + name
STOP

Example 3: Input for Calculation

START
    DECLARE num1, num2
    GET "Enter first number:", num1
    GET "Enter second number:", num2
    
    INIT sum = num1 + num2
    PRINT "The sum is: ", sum
STOP

Operator Precedence Examples

Operators are evaluated in a standard order (like PEMDAS). Use parentheses ( ) to force a specific order.

Example 1: Arithmetic Precedence

START
    // Multiplication * before Addition +
    INIT result1 = 10 + 5 * 2  // 10 + (5 * 2) = 20
    PRINT result1
    
    // Use parentheses to change order
    INIT result2 = (10 + 5) * 2  // (10 + 5) * 2 = 30
    PRINT result2
STOP

Example 2: Logical Precedence (AND, OR)

START
    // AND is evaluated before OR
    INIT result = TRUE OR FALSE AND FALSE // TRUE OR (FALSE AND FALSE) = TRUE
    PRINT result
STOP

Example 3: Logical Precedence (NOT)

START
    INIT age = 20
    INIT hasLicense = FALSE
    
    // NOT is evaluated first
    IF age >= 18 AND NOT hasLicense THEN
        PRINT "You are old enough, but need a license."
    ENDIF
STOP

IF / ELSEIF / ELSE

Executes code blocks based on a boolean condition. Supports IF, ELSEIF (or ELSE IF), and ELSE. The entire structure must end with ENDIF.

Example 1: Full Grade Checker

START
    DECLARE score
    INPUT "Enter score:", score

    IF score >= 97 THEN
        PRINT "Grade: 1.0 - Excellent"
    ELSEIF score >= 88 THEN
        PRINT "Grade: 1.75 - Good"
    ELSEIF score = 75 THEN
        PRINT "Grade: 3.0 - Passing"
    ELSE
        PRINT "Grade: 5.0 - Failing"
    ENDIF
STOP

Example 2: Nested IF Statements

START
    INIT age = 20
    INIT hasTicket = TRUE
    
    IF age >= 18 THEN
        PRINT "Age verified."
        IF hasTicket == TRUE THEN
            PRINT "Welcome to the event!"
        ELSE
            PRINT "You need a ticket."
        ENDIF
    ELSE
        PRINT "You are too young."
    ENDIF
STOP

Example 3: Simple IF / ELSE

START
    DECLARE num
    GET "Enter a number:", num
    
    IF num == 0 THEN
        PRINT "The number is zero."
    ELSEIF num % 2 == 0 THEN
        PRINT "The number is even."
    ELSE
        PRINT "The number is odd."
    ENDIF
STOP

SWITCH Statement

Selects one of many code blocks to execute based on an expression. Uses C-style syntax: SWITCH (expression), CASE value:, DEFAULT:, and BREAK. Ends with ENDSWITCH.

Example 1: Basic SWITCH with BREAK

START
    INIT discount = 0
    INIT code = 'B'
    
    SWITCH (code)
        CASE 'A':
            SET discount = 0.0
            BREAK
        CASE 'B':
            SET discount = 0.1
            BREAK
        CASE 'C':
            SET discount = 0.2
            BREAK
    ENDSWITCH
    PRINT "Discount is: ", discount // 0.1
STOP

Example 2: Fall-through (Omitting BREAK)

START
    INIT day = 2
    SWITCH (day)
        CASE 1:
        CASE 2:
        CASE 3:
        CASE 4:
        CASE 5:
            PRINT "It's a weekday."
            BREAK
        CASE 6:
        CASE 7:
            PRINT "It's the weekend!"
            BREAK
    ENDSWITCH
STOP

Example 3: Using DEFAULT

START
    DECLARE choice
    GET "Enter (1)Start or (2)Quit:", choice
    
    SWITCH (choice)
        CASE 1:
            PRINT "Starting..."
            BREAK
        CASE 2:
            PRINT "Quitting..."
            BREAK
        DEFAULT:
            PRINT "Invalid choice."
    ENDSWITCH
STOP

FOR Loop

Repeats a block of code a fixed number of times. Uses a counter variable. Optional STEP defines the increment. Ends with NEXT or ENDFOR.

Example 1: Count Up

START
    INIT i
    FOR i = 1 TO 3
        PRINT i
    NEXT // Prints 1, 2, 3
STOP

Example 2: Count Down (Negative STEP)

START
    INIT i
    FOR i = 5 TO 1 STEP -1
        PRINT i
    ENDFOR
    PRINT "Blast off!"
STOP

Example 3: Skip Counting (STEP 2)

START
    INIT i
    // Print all even numbers under 10
    FOR i = 0 TO 10 STEP 2
        PRINT i
    ENDFOR // Prints 0, 2, 4, 6, 8, 10
STOP

WHILE Loop

Repeats as long as a condition is TRUE. The condition is checked *before* the loop runs. Ends with ENDWHILE.

Example 1: Simple Counter

START
    INIT count = 0
    WHILE count < 3
        PRINT count
        count = count + 1
    ENDWHILE // Prints 0, 1, 2
STOP

Example 2: User Input Validation

START
    INIT choice = ""
    WHILE choice != "yes" AND choice != "no"
        INPUT "Continue? (yes/no)", choice
    ENDWHILE
    PRINT "You chose: " + choice
STOP

Example 3: Countdown

START
    INIT countdown = 3
    WHILE countdown > 0
        PRINT countdown
        countdown = countdown - 1
    ENDWHILE
    PRINT "Go!"
STOP

REPEAT UNTIL Loop

Repeats a block of code *until* a condition becomes TRUE. The block always runs at least once, as the condition is checked at the *end*.

Example 1: Input Validation

START
    DECLARE num
    REPEAT
        GET "Enter a number > 10:", num
    UNTIL num > 10
    PRINT "Finally! You entered ", num
STOP

Example 2: Simple Counter

START
    INIT count = 1
    REPEAT
        PRINT count
        count = count + 1
    UNTIL count > 5 // Prints 1, 2, 3, 4, 5
STOP

Example 3: Loop Runs Once

START
    INIT x = 10
    REPEAT
        PRINT "This runs once."
    UNTIL x == 10
STOP

Exiting Loops with BREAK

The BREAK keyword can be used to exit any loop (`FOR`, `WHILE`, `REPEAT`) immediately.

Example 1: BREAK in a FOR loop

START
    INIT i
    FOR i = 1 TO 10
        PRINT i
        IF i == 3 THEN
            PRINT "Breaking at 3"
            BREAK
        ENDIF
    ENDFOR // Prints 1, 2, 3, "Breaking at 3"
STOP

Example 2: BREAK in a WHILE loop

START
    INIT count = 0
    WHILE TRUE // An "infinite" loop
        PRINT count
        count = count + 1
        IF count >= 5 THEN
            BREAK
        ENDIF
    ENDWHILE // Prints 0, 1, 2, 3, 4
STOP

The compiler checks your code in two passes. Errors can be **Compile-Time** or **Run-Time**.

  • Compile-Time Error (Syntax): Your code is written incorrectly.
    • e.g., PRINT "Hello (unclosed string)
    • e.g., IF x > 5 (missing THEN)
    • e.g., PRINT "Hello" "World" (missing + operator)
  • Compile-Time Error (Block): Your code structure is wrong.
    • e.g., IF ... ENDWHILE (mismatched blocks)
    • e.g., ELSE ... ENDIF (missing IF)
  • Run-Time Error: Your syntax is valid, but the logic failed.
    • e.g., SET x = 10 (when x was not declared)
    • e.g., PRINT 5 / 0 (division by zero)
    • e.g., Potential infinite loop detected.
Tip: The red line highlight shows exactly where the error occurred. Read the terminal message for details.
