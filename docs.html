PseudoPlay allows you to write and execute pseudocode in a simple, intuitive environment. All programs must start with START (or BEGIN) and end with STOP (or END).

Comments

Comments help explain your code and are ignored during execution. Use //, #, --, or the COMMENT keyword for single-line comments, or /* ... */ for multi-line comments.

START

    // Single-line comment
    # Another single-line
    -- Yet another
    COMMENT This is also a comment
    
    /*
      Multi-line
      comment
    */
    
    PRINT "Hello, World!"
    
STOP
Tip: Use comments liberally to make your code readable. Press Ctrl+Enter to run your code quickly.

Variables store data like numbers, strings, or booleans. Declare them with DECLARE (initializes to null) or INIT (declare and assign). Data types are inferred automatically.

Declaring Variables

START
    DECLARE name  // null by default
    INIT age = 25
    INIT message = "Hello"
    INIT initial = 'J'
    INIT isActive = TRUE
    
    // Multiple declarations
    DECLARE x, y, z
    INIT a = 1, b = 2
STOP

Assigning Values

Use SET, LET, or direct assignment (=) to change values. Variables must be declared first.

START
    INIT score = 0
    SET score = 10
    LET score = score + 5
    score = 20  // Direct assignment
    
    PRINT score  // 20
STOP
Warning: Using undeclared variables will throw an error. Always use DECLARE or INIT before use.

Output (PRINT, DISPLAY, OUTPUT)

Display text, variables, or expressions. All three keywords work identically. You can combine multiple items with + or commas.

START
    INIT name = "User"
    PRINT "Hello, " + name + "!"
    OUTPUT "Score: ", 100
    DISPLAY 'A'
STOP

Input (INPUT, READ, GET)

Read user input and store it in a declared variable. All three keywords work identically. You can optionally provide a prompt.

START
    DECLARE age
    PRINT "Enter your age:"
    INPUT age
    
    DECLARE name
    READ "What is your name?", name
    
    PRINT "Hello " + name + ", you are " + age
STOP

IF Statements

Use IF for decisions. Supports ELSEIF (or ELSE IF) and ELSE. End with ENDIF.

START
    DECLARE score
    INPUT "Enter score:", score
    
    IF score >= 90 THEN
        PRINT "A Grade"
    ELSEIF score >= 70 THEN
        PRINT "B Grade"
    ELSE
        PRINT "Needs Improvement"
    ENDIF
STOP

SWITCH (C-Style)

Select one of many code blocks to execute. Uses C-style syntax with SWITCH (expression), CASE value:, and BREAK.

START
    DECLARE discount
    DECLARE code = 'B'
    
    SWITCH (code)
        CASE 'A':
            SET discount = 0.0
            BREAK
        CASE 'B':
            SET discount = 0.1
            BREAK
        CASE 'C':
            SET discount = 0.2
            BREAK
        DEFAULT:
            SET discount = 0.3
    ENDSWITCH
    
    DISPLAY "The discount is: ", discount
STOP
Fall-through Behavior: If you omit BREAK, execution will "fall through" and run the code in the *next* case block, just like in C or JavaScript.

FOR Loop

Repeat a fixed number of times. Optional STEP. End with NEXT or ENDFOR.

START
    INIT i
    FOR i = 1 TO 5 STEP 1
        PRINT i
    NEXT // or ENDFOR
STOP

WHILE Loop

Repeat while condition is true. End with ENDWHILE.

START
    INIT count = 0
    WHILE count < 5
        PRINT count
        count = count + 1
    ENDWHILE
STOP

REPEAT UNTIL Loop

Repeat until condition is true (runs at least once).

START
    INIT num = 0
    REPEAT
        PRINT "Enter >10:"
        INPUT num
    UNTIL num > 10
STOP

Exiting Loops with BREAK

The BREAK keyword can be used to exit any loop (`FOR`, `WHILE`, `REPEAT`) immediately.

START
    INIT i
    FOR i = 1 TO 10
        PRINT i
        IF i == 5 THEN
            PRINT "Breaking at 5"
            BREAK
        ENDIF
    ENDFOR
STOP
Warning: Avoid infinite loops! There's a max iteration limit to prevent crashes.

The following keywords are recognized for syntax highlighting but are **not yet implemented** by the interpreter. Using them will cause an "unimplemented" error.

  • FUNCTION / ENDFUNCTION
  • PROCEDURE / ENDPROCEDURE
  • CALL / RETURN
  • ARRAY / FOREACH / LENGTH
  • CONTINUE
FUNCTION Add(a, b)
    RETURN a + b
ENDFUNCTION
Note: Calling procedures/functions will error. Full support is coming in future updates.
  • Use consistent indentation (auto-indent helps!).
  • Declare all variables at the top of your program.
  • Add comments for complex logic.
  • Test small parts of code incrementally.
  • Avoid deep nesting; use functions when available.
  • Undeclared Variable: Always use DECLARE/INIT first.
  • Mismatched Blocks: Ensure IF/ENDIF, FOR/ENDFOR, SWITCH/ENDSWITCH, etc. all match.
  • Infinite Loop: Check loop conditions; max iterations prevent crashes.
  • Invalid Expression: Check your math and operators (e.g., `5 / 0` or `"a" * 2`).
  • Syntax Error: Check for missing quotes, colons (in `CASE`), or parentheses (in `SWITCH`).
Tip: The red line highlight shows exactly where the error occurred. Read the terminal message for details.
